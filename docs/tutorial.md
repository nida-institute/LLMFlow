# LLMFlow Quickstart Tutorial

This tutorial walks through the minimum working example that ships in the `hello/` folder. You will:

1. Generate the starter project with `llmflow init`
2. Inspect the scaffolded project structure
3. Understand the pipeline + prompt pairing
4. Add a second step that replies to each greeting
5. Run the pipeline with `llmflow run`
6. Customize and extend it

Everything below assumes you are inside the repository root (`LLMFlow/`).

> 📥 **Need the CLI?** Follow the platform-specific steps in [INSTALL.md](../INSTALL.md) to install the prebuilt executable before running the commands below.

---

## 0. Create the workspace with `llmflow init`

From the repo root:

```bash
llmflow init hello
cd hello
```

The init command creates a self-contained folder (pipelines, prompts, log, output). You can re-run it with a different target directory to bootstrap additional sandboxes.

---

## 1. Project Layout (freshly created)

```
hello/
├── llmflow.log          # run logs (auto-created)
├── output/              # pipeline outputs land here
├── pipelines/
│   └── hello-llmflow.yaml
└── prompts/
    └── hello.gpt
```

Key conventions:
- Pipelines live under `pipelines/`.
- Prompt templates live under `prompts/`.
- Output can be whatever directory you configure (here it is `output/`).
- `llmflow.log` is the consolidated log file (rotated per run).

---

## 2. Pipeline Anatomy (`hello/pipelines/hello-llmflow.yaml`)

```yaml
name: "Hello Multilingual Pipeline"
description: |
  Sample pipeline generated by `llmflow init`.
  Run it with:
    llmflow run --pipeline pipelines/hello-llmflow.yaml
variables:
  prompts_dir: "prompts"
  output_dir: "output"

llm_config:
  model: "gpt-4o-mini"
  temperature: 0.2
  max_tokens: 500

steps:
  - name: "multilingual-greeting"
    type: "llm"
    prompt:
      file: "hello.gpt"
      inputs:
        language_count: 5
    outputs: greeting
    saveas:
      path: "${output_dir}/hello.md"

  - name: "respond-to-greetings"
    type: "llm"
    prompt:
      file: "reply.gpt"
      inputs:
        greeting_markdown: "${greeting}"
    outputs: reply_block
    saveas:
      path: "${output_dir}/responses.md"
```

Highlights:
- `variables` expose paths that downstream steps can reuse (`${output_dir}` etc.).
- `llm_config` sets defaults for every `llm` step (model + sampling params).
- The first step points to `prompts/hello.gpt`, requests 5 languages, stores the response as `${greeting}`, then writes it to `output/hello.md`.
- The second step references `${greeting}` and uses `prompts/reply.gpt` to craft culturally appropriate responses, saving them to `output/responses.md`.

---

## 3. Prompt Anatomy (two files)

```gpt
---
system: |
  You are a friendly assistant who showcases multilingual greetings.
user: |
  Pick {{language_count}} languages randomly.
  For each one, identify the language in English, share a native-script greeting, and add a short cultural note.
  Respond as a markdown bullet list.
---
```

Notes:
- Prompt metadata is defined in YAML frontmatter (`system`, `user`, etc.).
- Templated variables use double curly braces (`{{language_count}}`).
- The `language_count` value comes from the pipeline step’s `inputs` block.

Add a second prompt file `prompts/reply.gpt` for the response step:

```gpt
---
system: |
  You are a cultural etiquette coach who replies warmly to multilingual greetings.
user: |
  You received this markdown list of greetings:

  {{greeting_markdown}}

  For each bullet, craft a short reply in the same language (when appropriate) plus an English explanation of the cultural nuance you appreciated. Return markdown with the same bullet structure.
---
```

This file expects the `greeting_markdown` string that the first step produced.

## 4. Run the Pipeline

From the `hello/` directory:

```bash
cd hello
llmflow run --pipeline pipelines/hello-llmflow.yaml
```

Behind the scenes:
1. LLMFlow validates the pipeline (structure, prompt contracts, variables).
2. It renders `prompts/hello.gpt` using `language_count: 5` and saves `output/hello.md`.
3. It feeds the resulting markdown into `prompts/reply.gpt`, generating culturally aligned replies in `output/responses.md`.
4. All interactions are logged to `llmflow.log`.

When the run finishes, open the output:

```bash
cat output/hello.md
cat output/responses.md
```

You should see something like:

```
- **Greek (Χαίρετε!)** – Greeting in Greek script...
- **Swahili (Hujambo!)** – ...
```

```
- **Reply to Greek greeting:** Χαίρετε κι από μένα! (Hello from me as well!) – Appreciated the warm communal tone…
- **Reply to Swahili greeting:** Hujambo pia! – Recognized East African hospitality…
```

Every run overwrites `output/hello.md`, so copy it elsewhere if you want to keep multiple versions.

## 5. Customizing the Example

| Change | Where | Notes |
| --- | --- | --- |
| Number of languages | `language_count` under `steps[0].prompt.inputs` | Any positive integer |
| Output path | `saveas.path` | Use `${variables}` or literal paths |
| Model | `llm_config.model` | Stick with models your API key supports |
| Prompt wording | `prompts/hello.gpt` | Update the `system` or `user` sections |
| Response behavior | `prompts/reply.gpt` | Adjust tone, languages, or add JSON output |

If you add more steps, remember that you can:
- Reference prior outputs via `${step_output}`
- Save intermediate artifacts with additional `saveas` blocks
- Use `function` steps for filesystem or transformation logic

---

## 6. Troubleshooting

| Symptom | Likely Cause | Fix |
| --- | --- | --- |
| `Prompt file not found` | `prompts_dir` or `file:` path incorrect | Double-check `variables.prompts_dir` and filename |
| `Missing required inputs` | Prompt contract requires a variable not provided | Ensure every value listed under `prompt.requires` exists in the step’s `inputs` |
| `llmflow run` fails immediately | API key not configured or model unavailable | Set provider auth env vars and confirm model access |
| Output file missing | `saveas.path` points to unwritable location | Keep paths inside repo or ensure parent dirs exist |
