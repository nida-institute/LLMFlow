# LLMFlow Document Index

This index tells the LLM where to look for detail.  
If you need deep info, ask for the document by name.

| Document | Focus |
|-----------|--------|
| **architecture.md** | System layout, engine modules, design patterns, and data flow. |
| **getting-started.md** | Installation, setup, and first-run instructions. |
| **GPT_CONTEXT.md** | Canonical AI-facing “voice” and style guide for model reasoning. |
| **llmflow-language.md** | Formal YAML spec: steps, vars, input/output binding, evaluation semantics. |
| **tutorial.md** | Minimal working example — define, run, and extend your first flow. |
| **why-LLMFlow.md** | The philosophy and motivation behind LLMFlow’s design choices. |

Use these when you need:
- **API / structure details** → `llmflow-language.md`
- **Conceptual grounding** → `why-LLMFlow.md`
- **Implementation specifics** → `architecture.md`
- **Instructional context** → `tutorial.md`, `getting-started.md`
- **Tone & persona guidance** → `GPT_CONTEXT.md`
